Self-Learning Text-to-SQL Engine personal project
Python · LangChain · LangGraph · SQL

You talk to it in plain English across a conversation — it handles follow-ups, ambiguous phrasing, references to what was said three messages ago. Under the hood it's a four-stage pipeline: intent recognition, contextual rephrasing, table selection, SQL generation. Each stage is its own component so you can swap one out without touching the rest. What makes it different is that it doesn't just generate SQL — it runs the query, checks if the result actually answers what you asked, then responds. And when it gets something wrong — bad table name, wrong column, schema mismatch — it writes the error to long-term memory. It won't make the same mistake twice. It gets better by failing.
